# Library CRM: Оптимизация книжного фонда 📚

Library CRM — это система управления сетью библиотек на базе Django. Главная фича проекта — интеллектуальный модуль «Оптимизация фонда». Система автоматически анализирует загруженность филиалов, вычисляет идеальные квоты с помощью математического метода Гамильтона и формирует точные логистические маршруты для перераспределения книг между библиотеками, защищая уникальные экземпляры.

## 🌟 Основной функционал

- Управление фондом и Дашборд
- Глобальная аналитика: Сводная статистика по всей сети (вместимость, количество книг на полках и на руках).
- Умный поиск: Единая строка поиска по инвентарным номерам, названиям книг, авторам, читателям и названиям филиалов.
- Изоляция N+1: Оптимизированные запросы к БД (select_related, prefetch_related, annotate), позволяющие обрабатывать тысячи книг без нагрузки на сервер.
- Интеллектуальное перераспределение
- Метод Гамильтона: Математически точный расчет целевого количества книг для каждого филиала на основе его физической вместимости.
- Защита выданных книг: Алгоритм никогда не предложит к вывозу книгу, которая в данный момент находится на руках у читателя.
- Защита уникальности (Анти-дубликат): При выборе книг на вывоз система в первую очередь забирает лишние копии (дубликаты), стараясь оставить в библиотеке хотя бы один экземпляр произведения.
- Умный Матчинг: Книги отправляются в те филиалы, где такого произведения еще нет ни на полках, ни в ожидаемых поставках.

## 🧠 Детальное описание логики распределения (Как это работает)

- *Алгоритм состоит из 4 последовательных шагов:*

1. Расчет квот: Вычисляется средний % загрузки сети. Каждая библиотека получает "целевое значение" (target) пропорционально своей вместимости.

2. Определение статусов: Те, у кого книг больше цели, становятся Донорами (Give). Те, у кого меньше — Реципиентами (Take).

3. Сбор пула: У Доноров с полок "изымаются" лишние книги (в приоритете дубликаты). Они помещаются в виртуальную корзину (donor_pool).

4. Матчинг (Приоритеты):
  - Приоритет 1: Отдать книгу туда, где её нет (максимальное разнообразие).

  -  Приоритет 2: Если книга есть у всех, отдать туда, где больше всего свободных мест.

**📊 Практический пример работы алгоритма**

Допустим, наша сеть состоит из трех библиотек:
```
Библиотека №1: Вместимость: 10 | Текущих книг: 5 (4 на руках, 1 на полке)

Библиотека №2: Вместимость: 8 | Текущих книг: 8 (0 на руках, полная загрузка, есть дубликаты)

Библиотека №3: Вместимость: 12 | Текущих книг: 5 (3 на руках)
```

**ШАГ 1: Базовая математика сети**
```
Общая вместимость = 30 (10 + 8 + 12)

Всего книг = 18 (5 + 8 + 5)

Средняя загрузка = 18 / 30 = 60%
```

**ШАГ 2: Расчет целей (Гамильтон)**
```
Умножаем вместимость на среднюю загрузку (60%):

Библиотека №1 (10 * 0.6) = Идеально 6 книг.

Библиотека №2 (8 * 0.6) = Идеально 4.8 книг -> округляем до 4, остаток 0.8.

Библиотека №3 (12 * 0.6) = Идеально 7.2 книг -> округляем до 7, остаток 0.2.

Распределение остатка: 6+4+7 = 17 книг. Осталась 1 нераспределенная книга. Она отдается библиотеке с наибольшим остатком (№2 с 0.8).

Финальные цели: №1 = 6, №2 = 5, №3 = 7.

**ШАГ 3: Выявление Доноров и Реципиентов**

Библиотека №1 (Цель: 6, Факт: 5). Статус: TAKE (нужна 1 книга).

Библиотека №2 (Цель: 5, Факт: 8). Статус: GIVE (нужно отдать 3 книги).

Библиотека №3 (Цель: 7, Факт: 5). Статус: TAKE (нужно 2 книги).
(Баланс сошелся: забираем 3, отдаем 1+2).
```
**ШАГ 4: Сбор и распределение**  
Система идет в Библиотеку №2.  
Ей нужно забрать 3 книги. На руках у читателей 0 книг, поэтому на полке доступны все 8. Система находит среди них дубликаты и забирает их в donor_pool.  
Затем система смотрит на Библиотеки №1 и №3 и распределяет эти 3 книги с учетом того, какие названия у них уже есть в базе. Книги на руках (у №1 их 4, у №3 их 3) алгоритм не трогает, но учитывает при проверке на дубликаты.

### Вариант А: Использование uv (Рекомендуется, очень быстро)

**Установка**  
```bash

uv --version

# Установка зависимостей
uv sync --no-dev

# Запуск проекта
uv run python manage.py runserver

# Остальные команды Django 
uv run <команда Django>
```
