# Library CRM: Оптимизация книжного фонда 📚

Library CRM — это система управления дублями книг в сети библиотек

## 🌟 Основной функционал

- Просмотр дэшборда по всем библиотекам с количетсвом книг и вместимостью.
- Просмотр всего фонда книг.
- Умный поиск: Позволяет искать библиотеку, либо книгу по названию ,либо по  автору.
- Создание книг и распределение их по библиотекам.
- Возможность зайти в админку.
- Просмотр аналитики с рекомендациями как поступить с дубликатами.
## 🚀 Быстрый старт

### Клонируйте репозиторий:

```bash
git clone git@github.com:aslan-py/library-project.git
cd library-project
```

### Вариант А: Использование uv (Рекомендуется, очень быстро)

**Установка**  
```bash
# Для Windows (PowerShell):
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# Для Linux/macOS:
curl -LsSf https://astral.sh/uv/install.sh | sh

# Проверка установки и версии, если есть версия то все - ok
uv --version

# Установка зависимостей
uv sync --no-dev

# Запуск проекта
uv run python manage.py runserver

# Остальные команды Django 
uv run <команда Django>
```

### Вариант Б: Использование стандартного pip

```bash
python -m venv venv
source venv/bin/activate   # Для Linux/macOS
pip install -r requirements.txt
```

### Примените миграции базы данных:

```bash
python manage.py migrate
```

### Загрузите тестовые данные (Фикстуры) для проверки работы алгоритма:

**Эта команда создаст 3 библиотеки, читателей и распределит книги (включая дубликаты)** 
``` bash
python manage.py loaddata initial_data.json
```

**Создать суперпользователя для доступа в админку. Можно свои дданыне создать.** 
``` bash
python manage.py createsuperuser
```

**Запуск сервера разработки:**

```
python manage.py runserver
```

Панель управления доступна по адресу: http://127.0.0.1:8000/


## 🧠 Детальное описание логики распределения (Как это работает) на примере вложенных фикстур.

1. Алгоритм смотрит на количество одинаковых книг и количество библиотек.  
`К примеру 4 книги и 3 библиотеки и все 4 в первой находятся.`
2. Делим количество книг на количетсво библиотек и получим справедливое распределение книг на каждую библиотеку  
`У нас получается по одной книге на каждую и одна остается лишняя`
3. Лишние книги распределяем слево направо по тем библиотекам где больше дублей именно этой книги  
`У нас все книги в первой библиотеке а занчит лишнюю добавляем к квоте первой библиотеке Итог: 1я :2 книги, 2я: 1 книга, 3я: 1 книга`
4. Смотрим каждую библиотеку на предмет совпадения с квотой и свободным капасити
`К примеру капасити у всех хватает.`
`Квота- 1я :2 книги, 2я: 1 книга, 3я: 1 книга`
`Реальное положение дел - 1я :4 книги, 2я: 0 книга, 3я: 0 книга`
`Распределение: 1я :2 книги(2 забираем), 2я: 1 книга(1 добавляем), 3я: 1 книга(1 добавляем)`
